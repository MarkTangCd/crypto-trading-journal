## Findings considered and rejected

These were surfaced during the audit but explicitly **not** turned into plans:

- **`transactions.list` returns `SELECT *`**: not worth doing at single-user
  scale; existing composite indexes (`transactions_account_status_idx`,
  `transactions_account_start_idx`, `transactions_account_end_id_idx`) already
  satisfy the hot queries.
- **Sidebar nav labels capitalized vs. DESIGN.md's lowercase rule**: chrome
  rather than data; author's design judgment call.
- **`transaction.update` zod input accepts close/review fields that the
  router then rejects**: server enforces the FSM correctly; the duplication
  is harmless, the tightening is cosmetic.
- **`pnpm.patchedDependencies` references `wouter@3.7.1` while deps pin
  `^3.3.5`**: project uses npm; pnpm metadata is inert.
- **Capitalized "Trading Journal" wordmark in DashboardLayout**: same chrome
  exception as above.
- **`vite-plugin-manus-runtime` and `client/public/__manus__/debug-collector.js`**:
  appear to be sandbox debug instrumentation, not application code; their
  removal is a separate, larger decision (does the project still run inside
  the Manus sandbox?). Plan 004 explicitly keeps them in scope.
