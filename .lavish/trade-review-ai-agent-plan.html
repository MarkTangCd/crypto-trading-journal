Trade Review · AI 复盘助手

在 Transactions 详情页打开一笔已完成的交易时，并列一位永远在线的复盘 agent。它先于你开口， 基于完整的交易上下文与开仓前后 100 根 K 线生成结构化分析；它可以接入互联网，主动查询当时的资讯与盘面成因； 它的能力随时间用 skill 扩展。它服务于一件事：把一笔已发生的交易，变成下一笔更好的交易。

目标

为什么要做

交易日志记录了 What，但很少回答 Why。 一位会读盘、能上网、记得你过往交易习惯的复盘搭子， 能让每一次「review」从填字变成对话。

已决议：v1 走 deepseek，对话 per-trade， web_search 后端选 tavily，Phase 5 4 个 skill 全做。

谁来用

一名独自交易、只服务自己的 trader。 不需要团队协作、不需要导出报告， 只需要在每天结束时被认真地、有据有理地追问几句。

什么算成功

打开一笔交易 → 自动得到一段有依据的初评 → 至少进行一轮反问， 然后在 Review Feedback 里写下被自己说服的句子。

现状 vs 目标

现状 · today

  • TransactionDetail 只有 Review Feedback 文本框 + Chart URL
  • 复盘完全靠脑补，缺少结构化提示
  • 没有 K 线对照、没有当时的新闻或链上信号
  • 历史交易记忆不可访问，无法做习惯总结
  • 没有任何 AI 入口
→

目标 · target

  • 右侧抽屉式 Agent 面板，默认折叠，一键展开
  • 打开即自动注入交易上下文，给出初始分析
  • K线窗口随提问扩展（before/after, finer TF）
  • Agent 可调用 web_search / web_fetch 检索盘面成因
  • 可在 Settings 切换 provider，新增 skill 即得新能力

Agent 上下文 · 自动注入到 system / first user message

trade 字段

tradingPair / direction / timeFrame / startTime / endTime / entryPrice / exitPrice / plannedStopLossPrice / plannedTakeProfitPrice / plannedRiskRewardRatio / positionSizeUsdt / outcome / returnAmount / tradingLogic / context / tradeItems / reviewFeedback。 全部直接从 transactions 表读出。

K 线窗口

复用 server/_core/coinank.ts · fetchCandles。 默认以 trade timeFrame 拉 entry 前 100 + 后 100 根， 序列化为 OHLCV JSON 注入。 边界规则：当 entry 时点距今不足 100 根（trade 刚发生）时，向后只取到最新一根即可， 不阻塞复盘；agent 上下文里附带实际命中根数让它知情。 额外提供 get_klines 工具，agent 可按需调更细 TF 或拉更宽窗口。

账户状态

initialBalance / currentBalance / consecutiveLosses / confidenceLevel（来自 server/db.ts 的计算字段）。 让 agent 知道这笔单子是处于「连亏后该谨慎」还是「连胜后该克制」的语境里。

历史样本

最近 N 笔同 pair / 同 direction 的交易摘要（精简到一行）， 用于回答"和我之前的入场比起来怎么样"这类问题， 而不让上下文爆炸。

模型 · 多 provider 抽象

所有 provider 走统一的 chat({ messages, tools, stream }) 接口； 差异在适配器内部消化，UI 永远不感知。

工具能力 · agent 主动调用

web_search
后端可替换 (Tavily / Serper / Brave Search)。 默认返回 top 5 摘要， agent 决定是否进一步 fetch。
network · token-light · cacheable
web_fetch
按 URL 抓取 → readability 抽正文 → markdown。 给 agent 看新闻原文、报告 PDF、Glassnode 帖子等。
network · 30s timeout · 200KB cap
get_klines
参数 (pair, timeFrame, anchor, before, after)， 复用 coinank，让 agent 自己缩放视野。
internal · already battle-tested
get_recent_trades
按 pair / direction / outcome 过滤的最近 N 笔历史交易， 用来回答习惯性问题。
internal · scoped to anonymous user

Skill 扩展机制 · 未来能力的入口

Skill = 一个本地 JS 模块，导出 { name, description, parameters, run }。 启动时扫描 server/agents/skills/*，注册为 agent 可见的工具集。 新增分析能力时，写一个文件即可，不动核心循环。

analyze_market_structure future skill · v0
输入一段 K 线，输出 swing high/low、趋势段、支撑阻力， 让 agent 用结构化语言代替"我觉得是个突破"。
fetch_funding_rates future skill · v0
拉 coinank funding rates，对照进出场时段， 判断是否在极端 funding 时段做反向。
compare_with_my_baseline future skill · v0
统计该用户最近 50 笔交易的 r:r 分布与胜率， 告诉 agent "这笔单子在你自己的样本里属于什么档次"。
on_chain_snapshot future skill · v0 (optional)
可选 skill，调用 glassnode / nansen 公开 endpoint， 仅在 pair = BTC/ETH 且时段命中时启用。

实施分阶段

01
骨架 · 单 provider 对话
TransactionDetail 右侧抽屉；trade 字段注入；非 streaming； 先跑通 deepseek（成本最低、协议最简）；conversations / messages 表落地。
~3 day
02
K 线上下文 + streaming
复用 coinank，自动拉 ±100 K 线；SSE streaming 渲染； 初始消息由 agent 自动生成（不需用户先开口）。
~2 day
03
多 provider + 设置页
ProviderAdapter 抽象；Settings 增 API key 输入 + 默认选择； per-conversation 可临时覆盖；密钥用本地 master key + node:crypto 加密。
~3 day
04
工具循环 · web_search / web_fetch / get_klines
实现 tool-calling 循环，最多 N 步保护；UI 渲染工具调用气泡； 搜索后端可热替换。
~4 day
05
Skill 注册中心
SkillRegistry boot-time 扫描；Settings 勾选启用； 先迁移现有内部工具到 skill 形态；为后续 analyze_market_structure 等留口。
~3 day

风险与失败模式

high
API key 安全
5 家 provider 的 key 都进 SQLite。即便单机，明文存储仍不可接受。 方案：node:crypto AES-GCM + 一份本地 master key 文件（os 权限 0600）， 首次启动自动生成；在 git ignore 中加入 master key 路径。
medium
上下文成本
200 根 K 线 + trade fields + history ≈ 6–12k tokens。 强制对 candles 进行 OHLCV 缩写、降精度；超过阈值时改用"摘要 + 工具拉细"模式。
medium
国内网络访问
gemini / openai 在国内不可直连。Provider 配置允许填 base URL 和 自定义 headers，方便用户走自己的反代；提供商错误友好降级到下一个可用 provider。
medium
工具调用死循环
硬上限：每次会话 max 12 步、单工具 max 5 次；时间预算 60s； 超出后强制 final answer，并在 UI 提示用户。
low
tool schema 跨 provider 差异
openai 系生态都吃 JSON Schema；gemini 走它自家的 FunctionDeclaration。 GeminiAdapter 内部做一次单向翻译即可，无需对外暴露差异。
low
单租户假设
tRPC context 仍走匿名用户。 agent_settings 全局；conversations 仍带 userId 字段以保留未来扩展余地。

已决议

default provider
deepseek
协议最简、国内可直连；Phase 1 即用它跑通端到端。
conversation scope
per-trade
一笔交易一段会话，关闭抽屉即归档，可重开；不引入"全局对话"概念。
web_search backend
tavily
v1 默认；适配器化保留切换 Serper / Brave 的能力。
phase 5 skills
4 个全做
analyze_market_structure · fetch_funding_rates · compare_with_my_baseline · on_chain_snapshot； 按依赖关系顺序实现，on_chain 仅 BTC/ETH 启用。
candle window 边界
后向自适应
entry 前 100 根硬要求；后向不足 100 根（trade 刚发生）取到最新一根即可， 实际命中根数同时告知 agent。
