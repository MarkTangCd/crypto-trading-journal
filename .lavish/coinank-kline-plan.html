decisions worth flagging

where does the fetch live?
server proxy via trpc. the project rule is “all api calls go through tRPC.” a server proxy also sidesteps any coinank cors header, lets us cache or rate-limit later, and keeps the upstream domain out of the browser network tab.
trpc
debounce length?
500 ms after last keystroke. short enough to feel responsive when the user pastes a pair, long enough to skip the typing path through btc → btcu → btcus. timeframe select fires immediately — it's a single click.
500ms
candle click flow?
modal, not silent fill. a single click on a candle today can land by accident; surfacing the four ohlc choices in a dialog makes the intent explicit and keeps the chart's existing marker behaviour intact.
modal
overwrite a typed entry price?
no warning, no block. the field stays freely editable. selecting from the dialog writes the field. an existing typed value gets replaced — same as if the user re-typed. matches the simplicity-first rule; no extra confirmation step.
replace
exchange parameter?
hard-coded Binance for now. coinank requires an explicit exchange. binance covers virtually all *USDT pairs the journal would use. surfacing an exchange dropdown is out of scope here.
binance
cache + react-query?
default tanstack settings. 100 candles is cheap; we leave the default stale time but disable refetchOnWindowFocus for this query so glancing at another tab doesn't repaint the chart while the user is filling fields.
no-focus-refetch
timeframe mapping?
map ui → api in one helper. the form's select already uses values like 1H, the api expects 1h; a single lookup table beats parsing on either side.
helper

server snippet · coinank.ts

// server/_core/coinank.ts
const COINANK_URL = "https://api.coinank.com/api/kline/list/open";

const INTERVAL_MAP: Record<string, string> = {
  "1m": "1m",  "5m": "5m",  "15m": "15m", "30m": "30m",
  "1H": "1h",  "4H": "4h",
  "1D": "1d",  "1W": "1w",  "1M": "1M",
};

export async function fetchCandles(input: {
  tradingPair: string;
  timeFrame:   string;
}): Promise<Candle[]> {
  const interval = INTERVAL_MAP[input.timeFrame];
  if (!interval) throw new TRPCError({ code: "BAD_REQUEST", ... });

  const url = `${COINANK_URL}?exchange=Binance`
    + `&symbol=${input.tradingPair.toUpperCase()}`
    + `&interval=${interval}&size=100&side=to&ts=${Date.now()}`;

  const res = await fetch(url, { headers: { accept: "application/json" } });
  if (!res.ok) throw new TRPCError({ code: "BAD_GATEWAY", ... });

  const body = await res.json();
  if (!body?.success || !Array.isArray(body.data)) return [];

  return body.data.map(mapRow);
}

risks & open questions

01
upstream is public & unauthed
no api key, no documented rate limit, no sla. server proxy gives us a single place to add a cache or back-off later without re-touching the client.
02
column order is o · c · h · l, not o · h · l · c
easy to flip by reflex — the mapping helper exists specifically to make the order explicit. a unit test on the mapper would be the cheapest insurance.
03
pair validation is permissive
we'll accept anything matching ^[A-Z0-9]{4,20}$ and forward to coinank; an unknown pair just yields an empty data array. inline “no data” message handles it without toast noise.
04
exchange is hard-coded
if a future pair only trades on okx or bybit, the chart will show empty for that pair. that's acceptable for v1 — a dropdown can be added later if it actually bites.
05
close-trade chart stays mocked
this task targets the newtransaction page only. the close-trade modal and any review surface keep their current mock data unless we agree to widen scope.
